## Purpose and Scope

This document explains the genesis block configuration and network type selection in StableNet. It covers the structure of genesis files, available network types (mainnet, testnet, devnet), chain configuration parameters, and the genesis generator utility. For information about running nodes on specific networks, see [Network Types and Deployment](/stable-net/go-stablenet/11.1-network-types-and-deployment). For node configuration flags, see [Node Configuration](/stable-net/go-stablenet/2.2-node-configuration).

## Genesis Block Overview

The genesis block is the first block in the blockchain, establishing the initial state of the network. It defines the chain configuration, initial account allocations, consensus parameters, and system contract deployments.

### Genesis Structure

The `Genesis` struct in [core/genesis.go58-77](https://github.com/stable-net/go-stablenet/blob/422cf168/core/genesis.go#L58-L77) defines the genesis block:

| Field | Type | Description |
| --- | --- | --- |
| Config | *params.ChainConfig | Chain configuration including fork blocks and consensus settings |
| Nonce | uint64 | Genesis block nonce |
| Timestamp | uint64 | Genesis block timestamp |
| ExtraData | []byte | Extra data field (contains WBFT validator info for Anzeon) |
| GasLimit | uint64 | Initial gas limit |
| Difficulty | *big.Int | Initial difficulty (1 for Anzeon) |
| Coinbase | common.Address | Genesis block coinbase address |
| Alloc | types.GenesisAlloc | Pre-allocated accounts and balances |
| BaseFee | *big.Int | EIP-1559 base fee |

### Production Network Genesis

Production networks require complete Anzeon configuration including all system contracts. The structure in [params/config.go44-143](https://github.com/stable-net/go-stablenet/blob/422cf168/params/config.go#L44-L143) demonstrates the required fields:

-   Complete chain configuration with all forks
-   WBFT parameters tuned for production
-   All five system contracts with proper initialization
-   Initial validator set with BLS keys
-   Bootstrap nodes for peer discovery

**Sources:** [params/config.go44-143](https://github.com/stable-net/go-stablenet/blob/422cf168/params/config.go#L44-L143)

## Summary

Genesis setup in StableNet involves:

1.  **Network selection** via command-line flags (`--mainnet`, `--testnet`, `--dev`)
2.  **Genesis validation** comparing stored and computed hashes
3.  **Chain configuration** defining fork blocks and consensus parameters
4.  **Anzeon initialization** creating validator extra data and deploying system contracts
5.  **Network parameters** including bootstrap nodes and chain IDs

The `genesis_generator` utility simplifies creating new networks, while the `SetupGenesisBlock` function handles initialization and validation during node startup.
